Technical reports with R Markdown & Bookdown

I provide markdown snippets for a variety of text environments and formats that can be created with R Markdown and the Bookdown package. I focus on the elements that I find most useful for writing math.

Combining R Markdown with functionality from the Bookdown package and Distill web publishing format provides a comprehensive set of tools for composing web documents. In this high-level reference guide I cover the tools that I find most useful for writing organized, easily referenced, math-heavy documents. The examples are arbitrary, only meant to illustrate the syntax. The LaTeX is not explained. For each example I provide the raw text followed by the rendered output.

1 Definition, theorem, proof

Ah, the familiar math textbook refrain. Below is the bookdown syntax for definition, theorem, and proof environments, respectively.

Definition

<change definition>

::: {#pyth-def .definition name="Pythagorean theorem"}
Given a right triangle, if $c$ denotes the length of the hypotenuse and
$a$ and $b$ the lengths of the other two sides, then
$$a^2 + b^2 = c^2$$.
:::

Definition 1 (Pythagorean theorem) Given a right triangle, if \(c\) denotes the length of the hypotenuse and \(a\) and \(b\) the lengths of the other two sides, then \[a^2 + b^2 = c^2.\]

Automatic numbering

The nice thing about bookdown is that it will automatically number definitions and theorems so that you can easily cross-reference them later on (see the <cross-ref-header> section below). Just replace the name following the # (pyth-thm in the example below) with a unique tag (containing no spaces) that will be convenient to reference.

<new definition>

::: {#pyth-thm2 .definition name="Pythagorean theorem"}
Given a right triangle, if $c$ denotes the length of the hypotenuse and
$a$ and $b$ the lengths of the other two sides, then
$$a^2 + b^2 = c^2$$.
:::

Definition 2 (Pythagorean theorem) Given a right triangle, if \(c\) denotes the length of the hypotenuse and \(a\) and \(b\) the lengths of the other two sides, then \[a^2 + b^2 = c^2.\]

Notice that the number given to this definition was incremented automatically from the previous definition environment.

4 Math blocks

Numbered blocks

Here is an example from the Bookdown documentation. It illustrates how to display a block of math with multiple lines that share a single number and tag. We use the LaTeX split environment to split the number across lines and wrap it all in double dollar signs to tell R to treat it as math rather than raw text.

$$
\begin{split}
\mathrm{Var}(\hat{\beta}) & =\mathrm{Var}((X'X)^{-1}X'y)\\
 & = (X'X)^{-1}X'\mathrm{Var}(y)((X'X)^{-1}X')'\\
 & = (X'X)^{-1}X'\mathrm{Var}(y)X(X'X)^{-1}\\
 & = (X'X)^{-1}X'\sigma^{2}IX(X'X)^{-1}\\
 & = (X'X)^{-1}\sigma^{2}
\end{split}
(\#eq:var-beta)
$$

\[ \begin{split} \mathrm{Var}(\hat{\beta}) & =\mathrm{Var}((X'X)^{-1}X'y)\\ & = (X'X)^{-1}X'\mathrm{Var}(y)((X'X)^{-1}X')'\\ & = (X'X)^{-1}X'\mathrm{Var}(y)X(X'X)^{-1}\\ & = (X'X)^{-1}X'\sigma^{2}IX(X'X)^{-1}\\ & = (X'X)^{-1}\sigma^{2} \end{split} \tag{2} \]

Numbered lines and inline comments

<Comment with && \text{(comment)}>

\[ \begin{align} \sum_{i=1}^{n} \left( X_i - \overline{X} \right ) & = \sum_{i=1}^{n}X_i - \sum_{i=1}^{n} \overline{X} &&& \text{(comment 1)} \tag{3} \\ & = \sum_{i=1}^{n} X_i - n \overline{X} &&& \text{(comment 2)} \\ & = \sum_{i=1}^{n}X_i - \sum_{i=1}^{n}X_i \notag \\ & = 0 \end{align} \]

5 Code blocks

<explain configuring Python interpreter>

## Display the Fibonacci sequence up to n-th term

nterms = int(input("How many terms? "))

# First two terms
n1, n2 = 0, 1
count = 0

# Check if the number of terms is valid
if nterms <= 0:
   print("Please enter a positive integer")

# If there is only one term, return n1
elif nterms == 1:
   print("Fibonacci sequence upto",nterms,":")
   print(n1)

# Generate fibonacci sequence
else:
   print("Fibonacci sequence:")
   while count < nterms:
       print(n1)
       nth = n1 + n2
       
       # update values
       n1 = n2
       n2 = nth
       count += 1
